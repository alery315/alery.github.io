windows更新引发的大坑
2019-01-02|个人博客|
Word count: 1,022|Reading time: 3 min

导火线

我在实验室电脑上装的是Ubuntu18.04+Windows10,这个Win10好像是原版的,不是那种ghost版的,而且还是个不稳定的版本,反正很奇怪,自动更新怎么都关不掉,试了无数种方法,都会自动恢复自动更新服务,就像是病毒一样的存在,正常更新就算了,它有个大的换版本更新,怎么都安装不成功,每次安装失败再进入Windows就会恢复,需要重启3次电脑才正常,下次又下载那个大更新.我实在受不了,想从跟本去抑制它,就改了注册表自动更新程序的路径,修改了,本来以为没事了,但是几天后这个注册表居然自己恢复了,又给我自动更新了.

事件经过

2019年一月一日下午,我来到实验室,又重启3次电脑,等了好一会,查了注册表,被自动恢复了,我佛了,照着路径,找到了c://Windows/system32/svchost.exe这个文件,就是执行更新的文件,右键删除,还不让我删!右键360强力删除,勾选防止文件再生,完美.重启电脑,想着它再也没法自动更新了,哈哈.然而,这次Windows都进不去了,赶紧查了下这个文件的作用,介绍如下:

Svchost.exe 是从动态链接库 (DLL) 中运行的服务的通用主机进程名称。这个程序对系统的正常运行是非常重要，而且是不能被结束的。许多服务通过注入到该程序中启动，所以会有多个进程。

惊了,我首先想到的是去别人win10那里拷一份,从Ubuntu系统挂载c盘,然后复制回去,然而Ubuntu挂载ntfs是写保护的,又搞了个ntfs权限修改的一个东西,没成功.

然后我就想着做个u盘启动吧,从pe进去,拷一份,这次是成功了,然而svchost这个文件,不仅是win7和win10对应的不一样,每个win10小版本好像都不一样,真的…

算了,下个ghost的win10重装吧,重装时候我选择的恢复系统,然后不写入引导,想着拿Ubuntu去修复引导方便点,然后重装完又是一片错误,具体:桌面是黑屏,然后什么都打不开,报什么错误我记不住了.然后再次重装,选择写入引导,这次Windows安装成功了,然后Ubuntu引导没了,拿easyBCD修复一下,成功跳出了熟悉的grub界面,然而后面还有一个大错误!

一直在紫色加载那里转圈,按esc查看,后台报错,ata1.00:failed to set xfermode(err_mask=0x4),然后有个runnning job在挂载一个盘,我的是/dev/sda7好像,我这里想到了之前为了复制文件win10,改了ntfs盘的权限好像,可能因为这个原因.

(在这里补充一步,我以为是引导被我覆盖了,做了一个Ubuntu启动盘,想要用boot-repair去修复,然后以失败告终.)

最后进入了emergency mode,GIVE root password for maintenance(or type ctrl+d to continue),输入密码,进入root命令行操作,vi /etc/fstab,如下图:

那个名字又长又奇怪的,还有C盘,都被我注释掉了,保存退出,reboot,完美,问题全部解决,Windows10也是正常的,Ubuntu也不用重装.

后记

之前遇到问题,不好解决的就直接重装系统了,但是最近一学期都在用Ubuntu,里面配置实在太多了,实在不想重装了,只能看错误,看log,查Google一步步解决了,对症下药.

就写这么多把,这学期基本都在做数据挖掘那个文本分类的实验,也没怎么写博客(记账流,- -),溜了,晚上还要考高级数理逻辑.(本来打算昨天1.1下午去实验室复习来着,结果一直弄到现在,才把电脑修好..)
